Login

The login process consists of the user entering a unique username and password pair. That when combined with a hidden salt and hashed using sha512.

This allows the testing of userd by checking for the existance of this hash.

On the checks success — that the hash is present in userd's memory — a session will be started on behalf of the user. This session consists of a timeout and a session token.

The timeout is used to automagically remove the session after a period of inactivity and the session token is used to continue the session as the user interacts with the web application. Both of these mechanisms can be implemented with a number of different pieces of software, the simplest would be using memcached to store sessions.



New Member

In creating a new member, we will generate a sha512 hash from the user submitted username and password. This hash will be used by userd for checking logins for that user.

In this way we reduce a user check and a password check into a single check against userd's in-memory collection of user hashes.

The use of userd as a in-memory store for accepted hashes allows the management of valid users to become seperated from the web application, and thus reusable as a service for future web applications.
